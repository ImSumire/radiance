Getting Started

To start a new project using the Radiance game framework, you can easily set it up by running the following command in your terminal:

curl https://imsumire.github.io/radiance/raylibc.sh | sh

This command downloads and executes a shell script that initializes the framework and sets up the necessary project structure.

Directory Breakdown

  • radiance/: Contains core files for the framework

    • extern.c: Used as main file for dlib
    • hot.h: Header for hot-reloading functionality
    • lib.h: Entry of Radiance
    • static.h: Used for release build

  • scripts/: Contains various scripts for project management

    • build.sh: Script to build the project
    • clean.sh: Script to clean up build artifacts
    • common/: Contains common scripts used across the project.
      • dlib.sh: Dynamic library management
      • hotcore.sh: Core hot-reloading functionalities
    • docs.sh: Script to open web documentations
    • hot.sh: Script to run the project with hot-reloading
    • init.sh: Script to initialize the project (automatically runned)
    • run.sh: Script to run the project as release
    • watch.sh: Script to watch for file changes and rebuild (used in hot.sh)

  • src/: Contains the source code of your game

    • common/: Common definitions and structures.
      • state.h: Contains the State struct, which holds the global state of the game.
    • lib.h: Contains base functions for initializing, updating, rendering, and cleaning up the game state.
    • main.c: The entry point of the game, containing the main game loop.

Running in Development Mode

To run your game in development mode with hot-reloading, execute the following command:

scripts/hot.sh

What is Hot-Reloading?

Hot-reloading is a feature that allows you to make changes to your code and see the results immediately without restarting the entire application. This is particularly useful during development as it speeds up the iteration process, allowing you to test changes quickly and efficiently. It helps maintain your workflow and reduces downtime, making it easier to experiment with game mechanics and features.

Building and Running in Release Mode

To build your project for release, use the following command:

scripts/build.sh

You can run the project (automatic build) with:

scripts/run.sh

The target executable will be located at target/linux/main. This is the compiled version of your game, ready for distribution or further testing.

Starting to Code Your Game

To start coding your game, you will primarily work within the src/ directory. Here are some steps to get you started:

  • Define Your Game State: Open src/common/state.h and define the State struct to hold your game's data. You can add fields such as player score, position, health, etc.
  • Implement Game Logic: Use lib.h to implement your game logic. The functions init, update, render, and drop are where you will define how your game initializes, updates each frame, renders graphics, and cleans up resources.
  • Modify the Main Loop: In main.c, you can modify the game loop to include your game logic. The loop will continuously call the update and render functions, so you can modifiy anything (not hot-reloaded).
